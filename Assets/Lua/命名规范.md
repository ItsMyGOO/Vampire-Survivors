# Lua 游戏项目命名规范（完整版）

> 适用范围
> - Lua 游戏逻辑
> - Lua 热更系统
> - 微信小游戏
> - Unity + XLua / Tolua
>
> 目标  
> **统一 · 可维护 · 可扩展 · 面试可讲**

---

## 一、总设计原则（必须遵守）

### 1. 统一优先于风格

- 一个统一但普通的规范  
  **强于**  
  多种“看起来更优雅”的混用

---

### 2. Lua 生态优先

- 遵循 Lua 官方库命名习惯
- 不照搬 C# / JavaScript 命名风格
- 逻辑与数据可读性优先

---

### 3. 数据 > 逻辑 > 表现

- 数据字段最严格
- 逻辑函数次之
- Bridge / 表现层允许有限对齐引擎

---

## 二、命名风格总览（全局）

| 类型 | 命名风格 | 示例 |
|---|---|---|
| 文件名 | PascalCase | `SkillPipeline.lua` |
| 模块 / 系统 | PascalCase | `BuffSystem` |
| 目录（系统级） | PascalCase | `Pipeline/Stage` |
| 函数名 | snake_case | `add_buff()` |
| 局部变量 | snake_case | `final_damage` |
| 成员字段 | snake_case | `cooldown_left` |
| Context / Config | snake_case | `skill_id` |
| Buff Hook | snake_case | `on_damage_calc` |
| 常量 / 枚举 | UPPER_SNAKE | `BUFF_TYPE` |

---
